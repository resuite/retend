# Server-Side Rendering

Normally, Single Page Applications load an empty HTML file first, and then rely entirely on JavaScript to draw the interface on the screen. This means users (and search engines) might see a blank page for a moment while the JavaScript downloads.

**Server-Side Rendering (SSR)** and **Static Site Generation (SSG)** fix this by building your HTML on the server before sending it to the browser. The user sees the full page instantly, and then Retend seamlessly attaches itself to make the page interactive.

## Static Site Generation

Retend provides a simple plugin for Vite that can automatically pre-render all of your routes into ready-to-serve HTML files when you build your project.

To use it, add `retendSSG` from `retend-server/plugin` into your `vite.config.ts`:

```ts
import path from 'node:path';
import { defineConfig } from 'vite';
import { retend } from 'retend-web/plugins/vite';
import { retendSSG } from 'retend-server/plugin';

export default defineConfig({
  resolve: {
    alias: { '@': path.resolve(__dirname, './source') },
  },
  plugins: [
    retend(),
    // Add the SSG plugin with the routes you want to build
    retendSSG({
      pages: ['/', '/about'],
      routerModulePath: './source/router.tsx',
      rootSelector: '#app',
    }),
  ],
});
```

When you run your build command, Retend will visit both the `/` and `/about` pages and generate static HTML files for them.

## Hydration

Once the browser loads that static HTML, Retend needs to "wake it up" and reconnect all your event listeners and Reactive Cells. This process is called **Hydration**.

Instead of using the standard `renderToDOM` function in your main client file, you use the `hydrate` function from `retend-server`:

```ts
import { hydrate } from 'retend-server/client';
import { createRouter } from './router';

// This brings the static HTML back to life!
hydrate(createRouter, {
  rootId: 'app',
});
```

In development mode, `hydrate` will act normally and render the app from scratch. But in production, it will look at the existing HTML and smartly attach to it without rebuilding the entire page.

## Handing Async Data

When rendering on the server, Retend needs to know when all your data has finished loading so it knows when to generate the HTML. It does this automatically using the `<Await>` boundaries you've already set up!

## Client-Only Components

Sometimes you have a component that uses browser-specific features (like reading the `window` size or accessing `localStorage`) that simply won't work on the server.

You can wrap these components in `<ClientOnly>`. They will be skipped during the server build, and will only render once the JavaScript loads in the user's browser:

```tsx
import { Await, Cell } from 'retend';
import { ClientOnly } from 'retend-server';

const asyncMessage = Cell.derivedAsync(async () => 'Ready');

export const App = () => (
  <Await fallback={<p>Loading...</p>}>
    <p>{asyncMessage}</p>

    {/* This button will NOT be rendered into the static HTML file */}
    <ClientOnly fallback={<span>Preparing interactive button...</span>}>
      <button type="button">Accesses Local Storage</button>
    </ClientOnly>
  </Await>
);
```

By safely isolating browser-only code, your application can render perfectly on the server every time.
