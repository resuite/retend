# Control Flow

In Retend, control flow is handled using pure JavaScript functions (`If`, `For`, and `Switch`) instead of React-style JSX conditionals or `Array.prototype.map`. This approach ensures type safety, better performance, and precise reactivity inside templates without the need for inline logical operators.

## The `If` Function

The `If` function is used for conditional rendering. It takes a condition (usually a `Cell`) and an object containing `true` and `false` branches.

```tsx
import { If, Cell } from 'retend';

function ToggleMessage() {
  const show = Cell.source(false);
  const toggle = () => show.set(!show.get());

  return (
    <div>
      <button onClick={toggle}>Toggle</button>

      {If(show, {
        true: () => <p>The message is visible!</p>,
        false: () => <p>The message is hidden.</p>,
      })}
    </div>
  );
}
```

If you only need to render something when the condition is true, you can omit the `false` branch:

```tsx
{
  If(show, {
    true: () => <p>Only shows when true.</p>,
  });
}
```

**Anti-Pattern Warning:** Do not use `&&` or ternary operators `? :` in JSX to conditionally render elements. Those operators do not bind reactively to Retend's DOM tracking and will break when Cells update.

## The `For` Function

The `For` function is used for rendering lists. It accepts an array or a reactive `Cell` containing an array, a mapping function, and an optional options object for key generation.

```tsx
import { For, Cell } from 'retend';

function ItemList() {
  const items = Cell.source([
    { id: 1, name: 'Apple' },
    { id: 2, name: 'Banana' },
  ]);

  return (
    <ul>
      {For(
        items,
        (item, index) => (
          <li>
            #{index}: {item.name}
          </li>
        ),
        { key: 'id' }
      )}
    </ul>
  );
}
```

### Keys in `For`

When working with arrays containing unique logical objects, you should always provide a `key` option to the `For` function. This key enables Retend to reuse DOM elements efficiently during sorts, additions, and deletions, instead of wiping and recreating the nodes.

You can provide:

1. A string representing the property name on the item to evaluate (e.g., `{ key: 'id' }`)
2. A function that takes the item and returns a string/number (e.g., `{ key: (item) => item.id }`)

Do not manually add `key={...}` props to the JSX elements returned from the callback. Retend handles DOM stability automatically using the `key` option block.

### Dynamic Children Parameters

Inside keyed `For` loops, sometimes the underlying array item properties might change without the `key` property changing. Rather than dropping reactivity to those sub-properties, map a derived cell to child components to retain granular tracking:

```tsx
{
  For(
    users,
    (user, index) => {
      const userCell = Cell.derived(() => users.get()[index.get()]);
      return <UserCard user={userCell} />;
    },
    { key: 'id' }
  );
}
```

_(Notice how `index` is a `Cell<number>`, so you must call `.get()` inside deriving contexts to access it!)_

## The `Switch` Function

The `Switch` function is an elegant way to handle multiple disjoint states, similar to a JavaScript `switch` block but returning node structures interactable in JSX.

```tsx
import { Switch, Cell } from 'retend';

function MultiState() {
  const status = Cell.source('loading'); // 'loading', 'success', or 'error'

  return (
    <div>
      {Switch(status, {
        loading: () => <p>Loading data...</p>,
        success: () => <p>Data loaded successfully!</p>,
        error: () => <p>Failed to load data.</p>,
      })}
    </div>
  );
}
```

If the string of `status` matches one of the property names in the object provided in the second argument, that specific node function is executed and attached to the DOM. If none match, it renders nothing.
