# Control Flow

Retend handles conditional rendering and lists differently than many frameworks. Instead of sprinkling logic operators throughout your JSX, you use pure JavaScript functions—`If`, `For`, and `Switch`—that keep your code type-safe, performant, and reactive.

## Conditional Rendering with `If`

Need to show different content based on a condition? The `If` function takes a Cell (or any value) and branches based on its truthiness.

```tsx
import { If, Cell } from 'retend';

function ToggleMessage() {
  const show = Cell.source(false);
  const toggle = () => show.set(!show.get());

  return (
    <div>
      <button type="button" onClick={toggle}>
        Toggle
      </button>

      {If(show, {
        true: () => <p>The message is visible!</p>,
        false: () => <p>The message is hidden.</p>,
      })}
    </div>
  );
}
```

You only need the `true` branch if that's all you're rendering:

```tsx
{
  If(show, {
    true: () => <p>Only shows when true.</p>,
  });
}
```

**Why not use `&&` or ternary operators?** Those operators don't bind to Retend's DOM tracking system, so they won't react when Cells change. Stick with `If` instead.

## Rendering Lists with `For`

The `For` function handles arrays—whether they're static or wrapped in a Cell. You give it the array, a callback that returns JSX for each item, and optionally a key strategy for efficient DOM reuse.

```tsx
import { For, Cell } from 'retend';

function ItemList() {
  const items = Cell.source([
    { id: 1, name: 'Apple' },
    { id: 2, name: 'Banana' },
  ]);

  return (
    <ul>
      {For(
        items,
        (item, index) => (
          <li>
            #{index}: {item.name}
          </li>
        ),
        { key: 'id' }
      )}
    </ul>
  );
}
```

### Using Keys for Efficiency

When you're rendering a list of objects with unique identifiers, always provide a `key` option. This tells Retend which DOM nodes can be reused when items are added, removed, or reordered—instead of destroying and recreating everything.

You can specify a key as a string (the property name):

```tsx
{
  key: 'id';
}
```

Or as a function that extracts the key:

```tsx
{
  key: (item) => item.userId;
}
```

Don't manually add `key` props to your JSX—Retend handles DOM stability automatically through the options object.

### Keeping Sub-Properties Reactive

Inside a keyed loop, if an item's properties change (but its key stays the same), you might lose fine-grained reactivity. To preserve it, derive a Cell for each item:

```tsx
{
  For(
    users,
    (user, index) => {
      const userCell = Cell.derived(() => users.get()[index.get()]);
      return <UserCard user={userCell} />;
    },
    { key: 'id' }
  );
}
```

`index` is a Cell, so call `.get()` when reading it inside derivations.

## Multi-State Rendering with `Switch`

When you have more than two states—like a data-fetching UI with "loading", "success", and "error" states—`Switch` is cleaner than nested `If` calls.

```tsx
import { Switch, Cell } from 'retend';

function DataFetcher() {
  const status = Cell.source('loading');

  return (
    <div>
      {Switch(status, {
        loading: () => <p>Fetching data...</p>,
        success: () => <p>Success!</p>,
        error: () => <p>Something went wrong.</p>,
      })}
    </div>
  );
}
```

`Switch` checks the Cell's value against each key in the object. When it finds a match, it runs that branch. If nothing matches, nothing renders.

---

These three functions are the foundation of Retend's control flow. They keep your conditionals and loops reactive and performant.
