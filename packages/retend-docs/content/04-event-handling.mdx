# Event Handling

Interactive applications need to respond to user actions like clicks, typing, and hovering. In Retend, listening to these events and updating your interface is straightforward and feels like writing standard JavaScript.

## Listening to Events

To listen for an event on a DOM element, you pass a function to a prop named after the event.

Unlike standard HTML which uses lowercase (`onclick`), Retend uses **camelCase** for event names, exactly like the DOM element properties you may be used to in vanilla JavaScript.

- HTML: `onclick="handleClick()"`
- Retend: `onClick={handleClick}`

Here is a simple example of a button that logs a message when clicked:

```tsx
function LoggingButton() {
  const handleClick = () => {
    console.log('Button was clicked!');
  };

  return (
    <button type="button" onClick={handleClick}>
      Click Me
    </button>
  );
}
```

You can pass a named function like `handleClick`, or you can define an inline arrow function directly inside the JSX:

```tsx
<button type="button" onClick={() => console.log('Clicked!')}>
  Click Me
</button>
```

## Updating State with Events

The most common reason to listen for an event is to update the state of your application. In Retend, this means calling the `.set()` method on your Cells.

Because Retend's component functions only run once, you don't have to worry about the component re-rendering when the handler runs. Retend simply updates the underlying UI elements directly.

```tsx
import { Cell } from 'retend';

function Counter() {
  const count = Cell.source(0);

  return (
    <div>
      <p>Current count: {count}</p>

      {/* Clicking the button updates the Cell's value */}
      <button type="button" onClick={() => count.set(count.get() + 1)}>
        Add +1
      </button>
    </div>
  );
}
```

## The Event Object

When your event handler function is called, Retend automatically passes the standard native DOM `Event` object as the first argument.

This gives you full access to everything you need to know about the event, such as the key that was pressed, the coordinates of a mouse click, or the current value of an input field.

```tsx
function SearchInput() {
  const handleInput = (event: Event) => {
    // We cast event.target to HTMLInputElement to access .value in TypeScript
    const target = event.target as HTMLInputElement;
    console.log('User typed:', target.value);
  };

  return (
    <div>
      <label for="search">Search:</label>
      <input type="text" id="search" onInput={handleInput} />
    </div>
  );
}
```

## Event Modifiers

In standard JavaScript event handling, you often find yourself writing boilerplate code right at the top of your function to manage the event's default behavior or propagation:

```tsx
// Standard JS boilerplate
const handleSubmit = (event: Event) => {
  event.preventDefault(); // Stop the page from reloading
  // ... actual logic ...
};
```

Retend provides **Event Modifiers** to handle these common scenarios directly in your JSX. You can append modifiers to the event name using a double hyphen (`--`).

```tsx
function Form() {
  const handleSubmit = () => {
    console.log('Form submitted securely!');
    // No need to call event.preventDefault() here!
  };

  return (
    <form onSubmit--prevent={handleSubmit}>
      <button type="submit">Submit Form</button>
    </form>
  );
}
```

### Available Modifiers

You can chain multiple modifiers together if needed (e.g., `onClick--prevent--stop={...}`).

- **`--prevent`**: Automatically calls `event.preventDefault()`. Essential for forms and links.
- **`--stop`**: Automatically calls `event.stopPropagation()`. Prevents the event from bubbling up to parent elements.
- **`--once`**: The event listener will only trigger the very first time the event occurs, and then immediately unbind itself.
- **`--passive`**: Marks the event listener as passive, improving scroll performance on events like `onScroll` or `onTouchMove` (cannot be used with `--prevent`).

```tsx
function EventShowcase() {
  return (
    <div onClick={() => console.log('Div clicked')}>
      {/* Prevents the form from reloading the page */}
      <form onSubmit--prevent={() => console.log('Form saved')}>
        <button type="submit">Save</button>
      </form>

      {/* Stops the click from reaching the parent <div> */}
      <button
        type="button"
        onClick--stop={() => console.log('Only button clicked')}
      >
        Stop Bubbling
      </button>

      {/* This will only log the very first time it is clicked */}
      <button type="button" onClick--once={() => console.log('Clicked once!')}>
        One-Time Click
      </button>
    </div>
  );
}
```

Event modifiers keep your handlers pure, focused strictly on your application logic rather than DOM mechanics. With event handling covered, you're ready to learn how to conditionally render content and display lists using Control Flow components.
