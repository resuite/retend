# Event Handling

When users click buttons, type in inputs, or hover over elements, Retend lets you respond with event props.

## Listening to Events

Adding event listeners in Retend is simple: pass a function to an event prop. Use **camelCase** event names (like `onClick`, `onInput`, `onSubmit`) rather than the lowercase HTML style.

```tsx
function LoggingButton() {
  const handleClick = () => {
    console.log('Button was clicked!');
  };

  return (
    <button type="button" onClick={handleClick}>
      Click Me
    </button>
  );
}
```

Prefer defining handlers before JSX and reusing them in event props:

```tsx
function ReusableButton() {
  const handleClick = () => {
    console.log('Clicked!');
  };

  return (
    <button type="button" onClick={handleClick}>
      Click Me
    </button>
  );
}
```

## Updating State from Events

The whole point of listening to events is usually to change something. In Retend, that means updating your Cells by calling `.set()`.

Since your component function only runs once, you don't have to worry about re-renders. Retend just updates the DOM elements directly:

```tsx
import { Cell } from 'retend';

function Counter() {
  const count = Cell.source(0);
  const increment = () => {
    count.set(count.get() + 1);
  };

  return (
    <div>
      <p>Current count: {count}</p>
      <button type="button" onClick={increment}>
        Add +1
      </button>
    </div>
  );
}
```

## Working with the Event Object

Your handler receives the standard DOM `Event` object, giving you access to whatever details you need: key presses, mouse coordinates, form values, and more.

```tsx
function SearchInput() {
  const handleInput = (event: Event) => {
    const target = event.target as HTMLInputElement;
    console.log('User typed:', target.value);
  };

  return (
    <div>
      <label for="search">Search:</label>
      <input type="text" id="search" onInput={handleInput} />
    </div>
  );
}
```

## Event Modifiers

Writing `event.preventDefault()` at the top of every form handler gets tedious. Retend lets you attach **modifiers** directly to your event names using double hyphens:

```tsx
function Form() {
  const handleSubmit = () => {
    console.log('Form submitted!');
    // preventDefault is handled automatically
  };

  return (
    <form onSubmit--prevent={handleSubmit}>
      <button type="submit">Submit</button>
    </form>
  );
}
```

### Available Modifiers

- **`--prevent`**: Calls `event.preventDefault()`. Essential for forms and links that would normally cause navigation.
- **`--stop`**: Calls `event.stopPropagation()`. Prevents the event from bubbling to parent elements.
- **`--once`**: The handler only fires once, then automatically unbinds itself.
- **`--passive`**: Marks the listener as passive, improving scroll performance on `onScroll` or `onTouchMove` (incompatible with `--prevent`).

You can combine modifiers if needed:

```tsx
function EventShowcase() {
  const handleParentClick = () => {
    console.log('Div clicked');
  };

  const handleFormSubmit = () => {
    console.log('Form saved');
  };

  const handleStopClick = () => {
    console.log('Only button clicked');
  };

  const handleOnceClick = () => {
    console.log('Just once!');
  };

  return (
    <div onClick={handleParentClick}>
      <form onSubmit--prevent={handleFormSubmit}>
        <button type="submit">Save</button>
      </form>

      <button type="button" onClick--stop={handleStopClick}>
        Stop Bubbling
      </button>

      <button type="button" onClick--once={handleOnceClick}>
        One-Time Click
      </button>
    </div>
  );
}
```

By handling event mechanics directly in your JSX, you can focus on your application logic instead of boilerplate.
