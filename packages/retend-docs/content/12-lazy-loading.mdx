# Lazy Loading

JavaScript bundles grow as applications grow. Ship a single massive file and users download, parse, and execute all of it—even the settings page they'll never visit. That's wasteful.

Retend solves this with code splitting. The `lazy()` function tells your bundler (like Vite) to split a route's component into its own file, downloaded only when the user actually navigates there.

## Basic Lazy Loading

Wrap a dynamic `import()` in `lazy()`. The imported module must have the component as its `default` export:

```tsx
import { defineRoutes, lazy } from 'retend/router';

const Home = () => <div>Home</div>;

// Loads on demand when user navigates to /settings
const SettingsLazy = lazy(() => import('./views/Settings'));

const routes = defineRoutes([
  { path: '/', component: Home },
  { path: '/settings', component: SettingsLazy },
]);
```

The initial JavaScript payload includes only `Home`. When a user clicks through to `/settings`, the router fetches the `Settings.js` chunk over the network and renders it seamlessly.

### Component Structure

When creating a file for lazy loading, remember: `lazy()` expects the component as the `default` export.

```tsx
// views/Settings.tsx

// Local helper components are fine
function SettingsForm() {
  return <form>...</form>;
}

// But the main component MUST be default
export default function SettingsView() {
  return (
    <div class="settings">
      <h1>Account Settings</h1>
      <SettingsForm />
    </div>
  );
}
```

## Lazy Nested Routes

Lazy loading works the same way with nested routes. You can lazily load the layout, its children, or both:

```tsx
import { defineRoutes, lazy, Outlet } from 'retend/router';

const DashboardLayout = () => (
  <div>
    <h1>Dashboard</h1>
    <Outlet />
  </div>
);

const UserProfileLazy = lazy(() => import('./views/dashboard/UserProfile'));

const routes = defineRoutes([
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [{ path: 'profile', component: UserProfileLazy }],
  },
]);
```

## Lazy Subtrees

For massive applications with hundreds of routes, even the routing config itself becomes too large. **Lazy Subtrees** let you split entire routing configurations and load them on demand.

Instead of a `component` or `children` array, use a `subtree` property that dynamically imports another routing file.

### Step 1: Create the Subtree File

Create a separate file that exports a single route configuration as its default export:

```tsx
// finance.routes.ts
import { defineRoute, lazy } from 'retend/router';
import FinanceDashboard from './views/FinanceDashboard';

// The path MUST match where it will be mounted in the parent router
export default defineRoute({
  path: '/finance/:accountId',
  component: FinanceDashboard,
  children: [
    {
      path: 'reports',
      component: lazy(() => import('./views/Reports')),
    },
    {
      path: 'invoices',
      component: lazy(() => import('./views/Invoices')),
    },
  ],
});
```

### Step 2: Connect in Main Router

In your main routes, use `subtree` with `lazy()`:

```tsx
// index.routes.ts
import { defineRoutes, lazy } from 'retend/router';

const routes = defineRoutes([
  { path: '/', component: Home },
  {
    path: '/finance/:accountId',
    // Entire subtree loads on demand
    subtree: lazy(() => import('./finance.routes.ts')),
  },
]);
```

With Lazy Subtrees, you can build infinitely scalable SPAs without sacrificing initial load performance.
