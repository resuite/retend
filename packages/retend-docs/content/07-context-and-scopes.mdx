# Context and Scopes

As applications grow, you inevitably face a common challenge: passing data down through many layers of components that don't actually need it themselves. A theme setting lives at the top level but needs to reach a button five levels deep. A user's authentication status belongs everywhere but doesn't make sense as a prop chain. This pattern—shoving data through every intermediate component—is called **prop drilling**, and it makes codebases painful to maintain.

Retend solves this elegantly with **Context Scopes**. They let you broadcast data from a parent so any descendant can grab it directly, completely skipping the middle layers.

## The Problem: Prop Drilling

Picture this structure:

```tsx
function App() {
  const theme = 'dark';
  return <Layout theme={theme} />;
}

function Layout(props) {
  return <Sidebar theme={props.theme} />;
}

function Sidebar(props) {
  return <Navigation theme={props.theme} />;
}

function Navigation(props) {
  return (
    <Link theme={props.theme} href="/">
      Home
    </Link>
  );
}
```

Only the `Link` actually uses the theme, but it has to travel through three components that don't care about it at all. Now imagine adding `userRole`, `locale`, and `isDarkMode` to that chain. Your component signatures become noise, and refactoring becomes a nightmare.

## Creating and Using Scopes

A Scope is how you establish a "broadcast channel" for data. Create one using `createScope()`:

```tsx
import { createScope } from 'retend';

export const ThemeScope = createScope('Theme');
```

The string is just for debugging in developer tools. What you get back is a Scope object that contains a `Provider` component.

To make data available, wrap your component tree in the Provider and pass your data to its `value` prop:

```tsx
import { ThemeScope } from './scopes';

function App() {
  const currentTheme = 'dark';

  return (
    <ThemeScope.Provider value={currentTheme}>
      <MainLayout />
    </ThemeScope.Provider>
  );
}
```

Now any component inside that Provider can access the theme by calling `useScopeContext()`:

```tsx
import { useScopeContext } from 'retend';
import { ThemeScope } from './scopes';

function ThemedButton() {
  const theme = useScopeContext(ThemeScope);
  return (
    <button
      type="button"
      class={[
        'btn',
        { 'btn-dark': theme === 'dark', 'btn-light': theme === 'light' },
      ]}
    >
      Click Me
    </button>
  );
}
```

No props needed. The theme travels directly from `App` to `ThemedButton`, regardless of how many components sit between them.

### What Happens Without a Provider?

If a component calls `useScopeContext(ThemeScope)` but no Provider exists above it, Retend throws a clear error.

## Why Not Just Use Global Variables?

You might wonder: "Retend components run once. Why can't I just create a Cell in a module and import it everywhere?"

```tsx
// store.ts
export const globalTheme = Cell.source('dark');

// Button.tsx
import { globalTheme } from './store';
```

This works for simple cases, but Scopes unlock two critical architectural benefits: **isolation** and **automatic cleanup**.

### Isolation and Reusability

A global Cell is a singleton. There's only one instance for the entire application.

This means you can't render two independent instances of a complex, reusable component on the same page without them stepping on each other's data. If you have a form component that depends on global state and you render it twice—once for editing and once for creating—both forms will share the same state. They'll overwrite each other.

Scopes are tied to their Provider. Each Provider instance can have its own data:

```tsx
function Comparison() {
  return (
    <div class="side-by-side">
      <ThemeScope.Provider value="light">
        <MyForm />
      </ThemeScope.Provider>

      <ThemeScope.Provider value="dark">
        <MyForm />
      </ThemeScope.Provider>
    </div>
  );
}
```

Each form looks up to its nearest `ThemeScope.Provider` and gets its own independent value. Completely isolated.

### Automatic Cleanup

Global state lives for the entire application lifetime. That's sometimes useful (like for user session data), but it's wasteful for temporary UI state.

Say you have a multi-step checkout flow with form data:

```tsx
const CheckoutScope = createScope('Checkout');

export function CheckoutFlow() {
  const formData = Cell.source({ items: [], step: 1 });

  return (
    <CheckoutScope.Provider value={formData}>
      <CheckoutStep1 />
      <CheckoutStep2 />
      <CheckoutStep3 />
    </CheckoutScope.Provider>
  );
}
```

The `formData` Cell only exists while the checkout is on screen. When the user navigates away and `CheckoutFlow` unmounts, the Provider is destroyed. Since nothing references `formData` anymore, JavaScript's garbage collector immediately reclaims the memory. Fresh start next time. No manual cleanup required.

Global variables would hold that stale form data in memory indefinitely, creating both memory leaks and confusing UX bugs.

## Passing Cells as Context Values

You'll often want to pass a Cell (not just a static value) through a Scope so child components can both read and update shared state:

```tsx
import { Cell } from 'retend';
import { createScope, useScopeContext } from 'retend';

const UserScope = createScope('User');

function App() {
  const user = Cell.source({ name: 'Alice', role: 'admin' });

  return (
    <UserScope.Provider value={user}>
      <Dashboard />
    </UserScope.Provider>
  );
}

function UserProfile() {
  const userCell = useScopeContext(UserScope);
  const userName = Cell.derived(() => userCell.get().name);
  const handleRoleChange = () => {
    const current = userCell.get();
    userCell.set({ ...current, role: 'user' });
  };

  return (
    <div>
      <p>Hello, {userName}</p>
      <button type="button" onClick={handleRoleChange}>
        Change Role
      </button>
    </div>
  );
}
```

Now child components can read the current value with `.get()` and update it with `.set()`, all without any prop drilling.

---

Context Scopes keep your component trees clean and your state organized.
