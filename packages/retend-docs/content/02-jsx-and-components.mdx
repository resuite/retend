# JSX and Components

Retend uses **JSX** to describe the user interface and **Components** to organize those descriptions into reusable blocks.

## Understanding JSX

JSX is a syntax extension that allows you to write HTML-like markup directly inside your JavaScript files. It keeps your interface structure and logic in one place.

```tsx
const element = <h1>Hello, Retend!</h1>;
```

When Retend processes JSX, it transforms the markup into highly optimized operations for the active renderer (such as the browser DOM).

### Standard HTML Attributes

Because JSX compiles directly to standard renderer operations, you use regular HTML attribute names.

For CSS classes, use the standard `class` attribute:

```tsx
<div class="card-container">
  <p class="text-large">Welcome</p>
</div>
```

For form input labels, use the standard `for` attribute:

```tsx
<label for="username">Username</label>
<input type="text" id="username" />
```

### Boolean Attributes

Attributes like `checked` on checkboxes or `disabled` on buttons represent simple true or false states. If you include the attribute name without a value, it defaults to `true`:

```tsx
<input type="checkbox" checked />
<button type="button" disabled>Processing...</button>
```

### Embedding JavaScript Expressions

You can insert JavaScript variables and expressions directly into your JSX using curly braces `{}`.

```tsx
const userName = 'Alice';
const userAge = 28;

const profile = (
  <div class="profile">
    <h2>{userName}</h2>
    <p>Age: {userAge}</p>
  </div>
);
```

This also works for attribute values. For inline styles, pass a JavaScript object with camelCased property names:

```tsx
const alertBox = (
  <div style={{ backgroundColor: 'red', color: 'white' }}>
    Something went wrong!
  </div>
);
```

## Understanding Components

Components allow you to package UI elements into reusable units. In Retend, a component is a standard JavaScript function that returns JSX.

Component names must start with a **capital letter** (PascalCase). This distinguishes custom components from standard HTML tags.

```tsx
function Greeting() {
  return <div>Welcome to our application!</div>;
}
```

You can use your defined component anywhere in your JSX:

```tsx
function App() {
  return (
    <main>
      <h1>Home Page</h1>
      <Greeting />
    </main>
  );
}
```

### The Run-Once Architecture

A critical aspect of Retend's design is that **components run exactly once**.

When you use a component, Retend executes the function a single time to build the initial elements. The function will not run again.

Retend handles data updates through **Cells**, which update the specific DOM elements directly without re-running the component function. This eliminates unnecessary recalculations and provides exceptional performance.

### Passing Data with Props

You can customize components by passing data into them using **props** (properties). Props look identical to HTML attributes:

```tsx
<UserCard name="Alice" age={30} />
```

Inside your component, you receive these props as a single object. It is conventional to destructure them immediately:

```tsx
function UserCard(props) {
  const { name, age } = props;

  return (
    <div class="card">
      <h3>{name}</h3>
      <p>Age: {age}</p>
    </div>
  );
}
```

### Passing Content with the Children Prop

To use a component as a container or layout wrapper, use the special `children` prop. It captures any content placed between the component's opening and closing tags:

```tsx
function AlertBox(props) {
  const { children } = props;

  return <div class="alert-box">{children}</div>;
}

function App() {
  return (
    <AlertBox>
      <strong>Warning!</strong> Please check your inputs.
    </AlertBox>
  );
}
```

### Forwarding Props to Elements

When building reusable wrapper components (like a custom button), you often need to forward standard attributes (like `id` or `disabled`) to the underlying HTML element.

Use the rest operator (`...rest`) to gather extra props, and the spread syntax to apply them to the element:

```tsx
function PrimaryButton(props) {
  const { children, ...rest } = props;

  return (
    <button type="button" class="btn-primary" {...rest}>
      {children}
    </button>
  );
}

// Usage: The 'disabled' and 'id' attributes are passed directly to the <button>
const ui = (
  <PrimaryButton id="submit-btn" disabled>
    Submit Form
  </PrimaryButton>
);
```
