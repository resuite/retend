# JSX and Components

At the heart of building interfaces with Retend are two powerful, interrelated concepts: **JSX** for describing what things look like, and **Components** for organizing those descriptions into reusable pieces.

## JSX: Writing Markup as Code

JSX is a syntax that lets you write HTML-like markup directly inside JavaScript. Rather than bouncing between separate HTML files and JavaScript files, JSX collapses that boundary—your structure, logic, and behavior all live together.

```tsx
const element = <h1>Hello, Retend!</h1>;
```

When Retend processes this, it transforms it into highly optimized code that directly reaches into your renderer (whether that's the DOM in the browser, or something else entirely) and builds exactly what you need.

### Attributes and the HTML You Know

Since JSX compiles down to regular DOM operations, you use standard HTML attribute names. For styling with classes:

```tsx
<div class="card-container">
  <p class="text-large">Welcome</p>
</div>
```

For labels that reference form inputs:

```tsx
<label for="username">Username</label>
<input type="text" id="username" />
```

### Boolean Attributes

Attributes like `checked` on checkboxes or `disabled` on buttons represent simple true/false states. In Retend, just include the attribute name and it's automatically `true`:

```tsx
<input type="checkbox" checked />
<button type="button" disabled>Processing...</button>
```

### Embedding JavaScript

The real power emerges when you mix JavaScript into your markup using curly braces `{}`. Variables, expressions, calculations—whatever you need:

```tsx
const userName = 'Alice';
const userAge = 28;

const profile = (
  <div class="profile">
    <h2>{userName}</h2>
    <p>Age: {userAge}</p>
  </div>
);
```

This works in attributes too. For inline styles, pass an object with camelCased property names (`backgroundColor` instead of `background-color`):

```tsx
const message = 'Something went wrong!';

const alertBox = (
  <div style={{ backgroundColor: 'red', color: 'white' }}>{message}</div>
);
```

## Components: Building Blocks

While JSX lets you describe individual pieces of UI, **components** let you package them up into reusable units. A component is simply a JavaScript function that returns JSX.

Component names must start with a **capital letter** (PascalCase). That's how Retend knows the difference between a standard HTML element like `<div>` and something you've custom-built like `<UserProfile>`.

```tsx
function Greeting() {
  return <div>Welcome to our application!</div>;
}
```

Once defined, use it anywhere in your JSX:

```tsx
function App() {
  return (
    <main>
      <h1>Home Page</h1>
      <Greeting />
    </main>
  );
}
```

### The Run-Once Architecture

Here's something critical about Retend components: **they run exactly once**. When you use a component, Retend calls that function one single time to create the initial elements for your renderer. The function never runs again.

Updates happen through a different mechanism entirely—something called **Cells** (we'll explore this next). This run-once design is what gives Retend its exceptional performance. No unnecessary recalculations, no expensive re-renders. Just precise, surgical updates when data changes.

### Props: Customizing Components

Components become far more useful when you can customize them. Pass data in using **props** (short for properties), which look just like HTML attributes:

```tsx
<UserCard name="Alice" age={30} />
```

Inside your component, receive this data as a single object (conventionally named `props`) and destructure it immediately:

```tsx
function UserCard(props) {
  const { name, age } = props;

  return (
    <div class="card">
      <h3>{name}</h3>
      <p>Age: {age}</p>
    </div>
  );
}
```

#### The `children` Prop

Sometimes components act as containers or layouts for other content. The special `children` prop captures anything placed between a component's opening and closing tags:

```tsx
function AlertBox(props) {
  const { children } = props;

  return <div class="alert-box">{children}</div>;
}

function App() {
  return (
    <AlertBox>
      <strong>Warning!</strong> Please check your inputs.
    </AlertBox>
  );
}
```

#### Spreading Props to HTML Elements

If you're wrapping a standard HTML element (like a styled button), you'll often want to let users pass standard attributes like `disabled`, `id`, or `aria-label` straight through. Use the rest operator to gather remaining props and the spread operator to apply them:

```tsx
function PrimaryButton(props) {
  const { children, ...rest } = props;

  return (
    <button type="button" class="btn-primary" {...rest}>
      {children}
    </button>
  );
}

// Usage: 'disabled' and 'id' pass directly to the button
<PrimaryButton id="submit-btn" disabled>
  Submit Form
</PrimaryButton>;
```
