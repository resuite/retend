# Await

The `<Await>` component creates an asynchronous boundary in your application. It coordinates loading states across component subtrees, displaying a fallback UI while data loads from an API or while other async work is performed.

Unlike some frameworks that use error-throwing to trigger boundaries, Retend uses **Async Cells** paired with control flow components for a cleaner, more explicit approach.

## Understanding Async Cells

Before diving into `<Await>`, you need to know about Async Cells. They are created using `Cell.derivedAsync` and allow you to handle asynchronous operations reactively.

When you pass an Async Cell to a control flow component like `<If>` or `<For>`, Retend automatically registers it with the nearest parent `<Await>` boundary.

## Basic Usage

To use an asynchronous boundary, wrap your components in `<Await>` and provide a `fallback` element. Then, inside your components, create and use your Async Cells.

```tsx
import { Await, If, Cell } from 'retend';

function UserProfile() {
  // Create the Async Cell inside the component
  const userData = Cell.derivedAsync(async () => {
    const response = await fetch('https://api.example.com/user');
    return response.json();
  });

  return (
    <div class="user-profile">
      {/* If userData is pending, the parent Await boundary shows the fallback */}
      {If(userData, {
        true: (user) => (
          <div>
            <h2>{user.name}</h2>
            <p>{user.email}</p>
          </div>
        ),
      })}
    </div>
  );
}

export function App() {
  return (
    <main>
      <h1>Dashboard</h1>
      {/* Displays "Loading user data..." until userData resolves */}
      <Await fallback={<p>Loading user data...</p>}>
        <UserProfile />
      </Await>
    </main>
  );
}
```

When `userData` is still loading, the `<Await>` boundary catches the pending state and displays "Loading user data...". Once the fetch completes, the boundary renders its actual children, and the user's name and email appear.

## Multiple Async Operations

An `<Await>` boundary waits for _all_ async operations within its subtree to complete. If you have multiple Async Cells in the same boundary, the fallback stays visible until everything resolves.

```tsx
import { Await, If, For, Cell } from 'retend';

function DashboardContent() {
  const userData = Cell.derivedAsync(async () => {
    const res = await fetch('/api/user');
    return res.json();
  });

  const posts = Cell.derivedAsync(async () => {
    const res = await fetch('/api/posts');
    return res.json();
  });

  return (
    <div>
      {If(userData, {
        true: (user) => <h2>{user.name}'s Dashboard</h2>,
      })}

      <div class="posts">
        {For(posts, (post) => (
          <article>
            <h3>{post.title}</h3>
            <p>{post.content}</p>
          </article>
        ))}
      </div>
    </div>
  );
}

export function Dashboard() {
  return (
    <Await fallback={<p>Loading everything...</p>}>
      <DashboardContent />
    </Await>
  );
}
```

Both `userData` and `posts` must finish loading before the dashboard content appears. If you want them to load independently, you can wrap them in separate `<Await>` boundaries.

## Async Cells vs. Regular Cells

This distinction matters. A regular `Cell` can hold a Promise as its value, but it doesn't trigger `<Await>` boundaries:

```tsx
import { Cell } from 'retend';

// This WON'T trigger an Await boundary
const promiseCell = Cell.source(fetch('/api/data'));

// This WILL trigger an Await boundary
const dataCell = Cell.derivedAsync(async () => {
  const response = await fetch('/api/data');
  return response.json();
});
```

If you want a boundary to wait for an async operation, you must use `Cell.derivedAsync`.

## Manual Registration

If you are building custom components that handle async logic and need to hook into the `<Await>` boundary system manually, you can use the `useAwait` hook.

```tsx
import { useAwait } from 'retend';

function CustomAsyncComponent() {
  // Get the nearest Await context
  const awaitContext = useAwait();

  if (awaitContext) {
    // Register an Async Cell with the boundary
    // The boundary will wait for this cell to resolve
    awaitContext.waitUntil(myAsyncCell);
  }

  return <div>Custom content</div>;
}
```

This is rarely needed in typical applications, but it's available for advanced integrations.

## Server-Side Rendering (SSR)

`<Await>` boundaries are essential for server-side rendering. The server needs to wait for all async operations to complete before serializing the final HTML. You can use the `waitForAsyncBoundaries()` helper in your SSR pipeline:

```tsx
import { waitForAsyncBoundaries } from 'retend';
import { renderToString } from 'retend-server';

async function handleRequest(req, res) {
  // 1. Initial render triggers all async dependencies
  const dom = renderer.render(<App />);

  // 2. Wait for all boundaries to resolve
  await waitForAsyncBoundaries();

  // 3. Now serialize to HTML (loading states are resolved)
  const html = renderToString(dom, window);

  res.send(`<!DOCTYPE html>\n${html}`);
}
```

---

Async boundaries are how Retend handles the common pattern of "show a loading state until data arrives". Combined with Async Cells, they keep your component tree simple and your data flows explicit.
