# Defining Routes

Traditional web applications work one way: you click a link, the browser fetches a whole new HTML document, and the page reloads. It's simple but slow.

Single Page Applications (SPAs) take a different approach. The initial page loads once, and then JavaScript takes over to handle all subsequent navigation. URLs change, but the page doesn't reload. This creates a fast, app-like experience that feels native to the browser.

Retend is built for this model. It ships with a powerful router right out of the box—no third-party libraries needed.

## Setting Up Routes

To define your application's routing, you create an array of **Route Records**. Each record connects a URL path to a component.

While plain arrays work fine, the `defineRoutes` helper gives you better TypeScript inference and IDE autocompletion:

```tsx
import { Router, defineRoutes, createRouterRoot } from 'retend/router';
import { renderToDOM } from 'retend-web';

// Your page components
const Home = () => <div>Welcome to the Home Page</div>;
const About = () => <div>About Us</div>;

// Define your routes
const routes = defineRoutes([
  { path: '/', component: Home },
  { path: '/about', component: About },
]);

// Initialize and render
const router = new Router({ routes });
const root = document.getElementById('app')!;
renderToDOM(root, () => createRouterRoot(router));
```

The `createRouterRoot` function takes your router and turns it into a renderable component. More importantly, it establishes a **router context** that any component in your app can tap into using the `useRouter()` hook:

```tsx
import { useRouter } from 'retend/router';

function BackButton() {
  const router = useRouter();
  return <button onClick={() => router.back()}>Go Back</button>;
}
```

## Nested Routes and Layouts

Most apps have a consistent structure across pages—a header here, a sidebar there. Building this with flat routes means copy-pasting shared components into every page, which is both repetitive and wasteful (those components get destroyed and recreated on every navigation).

Nested routes solve this elegantly. A parent route acts as a layout, and the `<Outlet />` component marks where child routes should render:

```tsx
import { defineRoutes, Outlet } from 'retend/router';

function DashboardLayout() {
  return (
    <div class="dashboard-layout">
      <aside>Sidebar Navigation</aside>
      <main>
        <Outlet />
      </main>
    </div>
  );
}

const Overview = () => <h2>Dashboard Overview</h2>;
const Settings = () => <h2>User Settings</h2>;

const routes = defineRoutes([
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      { path: 'overview', component: Overview },
      { path: 'settings', component: Settings },
    ],
  },
]);
```

The `DashboardLayout` component stays mounted while navigating between its children. Only the content inside `<Outlet />` swaps out.

### Index Routes

What happens when a user visits `/dashboard` exactly? The Outlet would be empty. You can define a default child route by setting its path to an empty string:

```tsx
const routes = defineRoutes([
  {
    path: '/products',
    component: ProductsLayout,
    children: [
      { path: '', component: ProductList },
      { path: 'categories', component: CategoriesList },
    ],
  },
]);
```

Now `/products` shows the product list by default.

## Dynamic URL Segments

Some URLs need to capture variable data—a user ID, a post ID, a category name. Prefix a segment with a colon to make it dynamic:

```tsx
const routes = defineRoutes([
  // Matches /users/123 or /users/alice
  { path: '/users/:userId', component: UserProfile },

  // Multiple dynamic segments work too
  { path: '/posts/:category/:postId', component: BlogPost },
]);
```

How do you access these parameters in your components? That's covered in the **Queries and Params** section, but the short answer is the `useParams()` hook.

## Wildcards and Catch-Alls

### Matching Anything: The Catch-All Route

For custom 404 pages, use a single asterisk (`*`). Since routes are evaluated in order, put this at the end:

```tsx
const routes = defineRoutes([
  { path: '/', component: Home },
  { path: '/about', component: About },
  { path: '*', component: NotFoundPage },
]);
```

### Capturing Multiple Segments

Building something like a file explorer? You might need to capture an unknown number of trailing segments. Add an asterisk after a dynamic parameter:

```tsx
const routes = defineRoutes([
  // Matches /files/images/2026/vacation/photo.png
  // Captures "images/2026/vacation/photo.png" into filePath
  { path: '/files/:filePath*', component: FileViewer },
]);
```
