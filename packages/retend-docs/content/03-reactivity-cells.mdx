# Reactivity and Cells

In Retend, **components only run exactly once** to create the initial elements for the active renderer. The component functions themselves never run again.

To update the screen when data changes, Retend uses a precise, fine-grained reactivity system based on **Cells**.

---

## What are Cells?

A `Cell` is a reactive container for a value. When you put a piece of data inside a Cell, Retend can track exactly where that data is used in your application. When the data inside the Cell changes, Retend surgically updates only the specific parts of the renderer's tree that depend on it, without re-rendering anything else.

To create a new piece of reactive state, you use the `Cell.source()` function.

```tsx
import { Cell } from 'retend';

// Create a Cell holding the number 0
const count = Cell.source(0);
```

### Reading and Writing

To read the current value inside a Cell, you call its `.get()` method. To change the value, you call its `.set()` method.

```tsx
import { Cell } from 'retend';

const count = Cell.source(0);

// Reading the value (returns 0)
console.log(count.get());

// Writing a new value
count.set(5);

// Reading the updated value (returns 5)
console.log(count.get());
```

---

## Cells in JSX

The true power of Cells becomes apparent when you use them inside JSX.

When you want the UI to automatically update whenever a Cell changes, **you pass the Cell directly into your JSX**. You do _not_ call `.get()`.

```tsx
import { Cell } from 'retend';

function Counter() {
  const count = Cell.source(0);

  const increment = () => {
    // We use .get() here to read the current value and add 1
    count.set(count.get() + 1);
  };

  return (
    <div>
      {/* ❌ Incorrect: This only passes the static number 0 */}
      <p>Static count: {count.get()}</p>

      {/* ✅ Correct: We pass the Cell itself */}
      <p>Reactive count: {count}</p>

      <button onClick={increment}>Increment</button>
    </div>
  );
}
```

By passing the `count` Cell directly into the paragraph tag, Retend establishes a live connection. When `increment` is called and `.set()` updates the value, Retend immediately reaches into the renderer's tree and changes just that specific text node. The `Counter` function itself never runs again.

## Derived Cells

Often, you have data that depends on other pieces of data. For example, if you have a `firstName` Cell and a `lastName` Cell, you might want a `fullName` that automatically updates whenever either of the names change.

In Retend, you handle this using `Cell.derived()`.

```tsx
import { Cell } from 'retend';

function UserProfile() {
  const firstName = Cell.source('Alice');
  const lastName = Cell.source('Smith');

  // Create a derived cell that automatically tracks its dependencies
  const fullName = Cell.derived(() => {
    return `${firstName.get()} ${lastName.get()}`;
  });

  return (
    <div>
      <h1>{fullName}</h1>
    </div>
  );
}
```

### How Derived Cells Work

When you pass a function into `Cell.derived()`, Retend runs that function immediately. As the function runs, Retend watches to see which other Cells have their `.get()` method called. Retend automatically tracks these dependencies.

If `firstName` or `lastName` ever changes, Retend knows it needs to re-run the `fullName` derivation function to get the new value.

### Important Rules for Derived Cells

1. **No Dependency Arrays**: Unlike React hooks, you never have to manually specify an array of dependencies. Retend tracks them automatically.
2. **Read-Only**: Derived Cells are strictly for computing values. You can read them using `.get()` (or by passing them to JSX), but you can never call `.set()` on a derived cell.
3. **No Side Effects**: The function you pass to `Cell.derived()` should be pure. It should only return a value. You should never manually mutate the UI, trigger network requests, or call `.set()` on other Cells from inside a derivation.

---

## Side Effects

If you _do_ need to run arbitrary code whenever a Cell changes—such as manually mutating the UI, logging to the console, or syncing data to `localStorage`—you should use the `.listen()` method instead of creating a derived cell.

```tsx
import { Cell } from 'retend';

const theme = Cell.source('light');

// listen() runs the provided function whenever the theme Cell changes
theme.listen((newTheme) => {
  document.body.className = newTheme;
  localStorage.setItem('user-theme', newTheme);
});
```

The listener function will automatically be called with the new value every time the Cell's value is updated via `.set()`.
